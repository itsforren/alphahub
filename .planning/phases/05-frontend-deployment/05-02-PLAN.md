<task type="auto">
  <name>Task 2: Configure custom domain and Supabase Auth redirect URLs</name>
  <files>(no local files -- DNS and Supabase dashboard configuration)</files>
  <action>
  **Step 1: Add custom domain to Vercel**

  ```bash
  vercel domains add hub.alphaagent.io
  ```

  Vercel will provide a CNAME target (usually `cname.vercel-dns.com` or a project-specific hash like `d1d4fc829fe7bc7c.vercel-dns-017.com`). Note this value.

  If the CLI doesn't work, the user adds the domain in Vercel Dashboard > Project > Settings > Domains.

  **Step 2: Configure DNS at Hostinger**

  This step requires the user to add a CNAME record in the Hostinger DNS Zone Editor for alphaagent.io:

  ```
  Type:  CNAME
  Name:  hub
  Value: cname.vercel-dns.com  (or the specific value Vercel provided)
  TTL:   3600
  ```

  Check if the Hostinger MCP tool can do this. If available, use:
  - DNS_getDNSRecordsV1 to check current records for alphaagent.io
  - DNS_updateDNSRecordsV1 to add the CNAME record

  If the MCP tool is not available or fails, document the CNAME record for the user to create manually in Hostinger DNS Zone Editor.

  **Step 3: Verify DNS propagation**

  ```bash
  # Check DNS resolution (may take 5-30 minutes)
  dig hub.alphaagent.io CNAME +short
  # Expect: cname.vercel-dns.com. (or similar)

  # Once propagated, verify HTTPS works
  curl -s -o /dev/null -w "%{http_code}" https://hub.alphaagent.io
  # Expect: 200
  ```

  **Step 4: Configure Supabase Auth redirect URLs**

  Use the Supabase Management API to update auth redirect URLs:

  ```bash
  # Get current auth config
  curl -s -H "Authorization: Bearer $SUPABASE_ACCESS_TOKEN" \
    "https://api.supabase.com/v1/projects/qcunascacayiiuufjtaq/config/auth" | jq '.site_url, .redirect_urls'
  ```

  Update the auth configuration:
  ```bash
  curl -X PATCH \
    -H "Authorization: Bearer $SUPABASE_ACCESS_TOKEN" \
    -H "Content-Type: application/json" \
    "https://api.supabase.com/v1/projects/qcunascacayiiuufjtaq/config/auth" \
    -d '{
      "site_url": "https://hub.alphaagent.io",
      "uri_allow_list": [
        "https://hub.alphaagent.io/**",
        "http://localhost:8080/**",
        "http://localhost:5173/**"
      ]
    }'
  ```

  If the Management API doesn't support this, the user configures it in Supabase Dashboard > Authentication > URL Configuration:
  1. Set Site URL to `https://hub.alphaagent.io`
  2. Add `https://hub.alphaagent.io/**` to Redirect URLs allowlist
  3. Keep localhost entries for local development

  **Step 5: Verify auth config**

  ```bash
  curl -s -H "Authorization: Bearer $SUPABASE_ACCESS_TOKEN" \
    "https://api.supabase.com/v1/projects/qcunascacayiiuufjtaq/config/auth" | jq '{site_url, uri_allow_list}'
  ```

  Confirm `site_url` is `https://hub.alphaagent.io` and `uri_allow_list` includes `https://hub.alphaagent.io/**`.
  </action>
