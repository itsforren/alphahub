  **Step 1: Update .gitignore to exclude .env files**

  Add these lines to the EXISTING `.gitignore` file (append at the end):

  The current `.gitignore` already has `supabase/.env.secrets` but does NOT have `.env` itself. This is critical -- the `.env` file contains Supabase credentials.

  **Step 2: Create vercel.json for SPA routing**

  Create `vercel.json` in the project root:

  ```json
  {
    "rewrites": [
      {
        "source": "/(.*)",
        "destination": "/index.html"
      }
    ]
  }
  ```

  This ensures all routes (e.g., `/hub/admin/clients`) serve `index.html` so React Router handles them client-side. Without this, direct URL access or page refresh on any route except `/` returns Vercel's 404 page.

  **Step 3: Create .env.example as documentation**

  Create `.env.example` in the project root:

  ```
  # Supabase Connection
  VITE_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
  VITE_SUPABASE_PUBLISHABLE_KEY=your_anon_key_here
  VITE_SUPABASE_ANON_KEY=your_anon_key_here
  VITE_SUPABASE_PROJECT_ID=YOUR_PROJECT_REF
  ```

  Note: Both VITE_SUPABASE_PUBLISHABLE_KEY and VITE_SUPABASE_ANON_KEY must be set to the same value. The codebase uses both names in different files.
