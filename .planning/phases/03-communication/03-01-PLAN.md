    **ChatModels.swift** -- Create Codable structs matching the existing Supabase tables:

    1. `ChatConversation`: Codable, Identifiable, Sendable. Fields: id (String), clientId (String), adminId (String?), lastMessageAt (String?), unreadCountClient (Int), unreadCountAdmin (Int), createdAt (String). CodingKeys for snake_case mapping.

    2. `ChatMessage`: Codable, Identifiable, Sendable. Fields: id (String), conversationId (String), senderId (String), senderName (String), senderRole (String -- "client" or "admin"), senderAvatarUrl (String?), message (String), attachmentUrl (String?), attachmentType (String?), attachmentName (String?), linkPreview (LinkPreviewData?), readAt (String?), createdAt (String). CodingKeys for snake_case mapping. Computed property `isClient: Bool` returns senderRole == "client". Computed property `parsedCreatedAt: Date?` uses ISO8601 parsing (same pattern as DateFormatting.swift).

    3. `LinkPreviewData`: Codable, Sendable. Fields: url (String?), title (String?), description (String?), image (String?), siteName (String?).

    4. `ChatMessageInsert`: Codable, Sendable. For inserting new messages. Fields: conversationId, senderId, senderName, senderRole, senderAvatarUrl, message. CodingKeys for snake_case.

    5. `PendingMessage`: Identifiable (NOT Codable). Fields: id (UUID), text (String), status (enum: sending, failed), createdAt (Date). This is for local-only offline queue display.

    6. `ChatSettings`: Codable, Sendable. Fields: id (String?), businessHoursStart (Int), businessHoursEnd (Int), businessHoursTimezone (String), businessDays ([String]). CodingKeys for snake_case.

    **RealtimeManager.swift** -- Create `@Observable @MainActor final class RealtimeManager`:

    - Properties: `isConnected: Bool`, `typingUsers: Set<String>`, private `channel: RealtimeChannelV2?`, private `messageTask: Task<Void, Never>?`, private `typingTask: Task<Void, Never>?`, private `supabase: SupabaseClient`
    - `init(supabase:)` with default `SupabaseConfig.client`
    - `connect(conversationId: String, onNewMessage: @escaping (ChatMessage) -> Void) async`:
      - Call `disconnect()` first to clean up
      - Create channel: `supabase.channel("chat-\(conversationId)")`
      - Subscribe to postgres_changes: `ch.postgresChange(InsertAction.self, schema: "public", table: "chat_messages", filter: .eq("conversation_id", value: conversationId))`
      - Subscribe to broadcast: `ch.broadcastStream(event: "typing")`
      - `await ch.subscribe()`
      - Set `isConnected = true`
      - Start `messageTask` to iterate insertions, decode as ChatMessage, call `onNewMessage`
      - Start `typingTask` to iterate typing broadcast, parse user_id and is_typing from payload, update `typingUsers` set, auto-clear after 5 seconds
    - `sendTyping(isTyping: Bool) async`: Broadcast typing event on channel
    - `disconnect() async`: Cancel tasks, remove channel via `supabase.removeChannel()`, reset state
    - `reconnect(conversationId:onNewMessage:) async`: Just calls `connect()` (clean reconnect)

    **ChatViewModel.swift** -- Create `@Observable @MainActor final class ChatViewModel`:

    - Properties: `messages: [ChatMessage]`, `pendingMessages: [PendingMessage]`, `isLoadingInitial: Bool`, `isLoadingMore: Bool`, `hasMoreMessages: Bool`, `conversationId: String?`, private `knownMessageIds: Set<String>`, private `nextCursor: String?`, private `supabase: SupabaseClient`
    - `init(supabase:)` with default
    - `loadConversation(clientId: String) async`: Get or create conversation (try select where client_id = clientId, if nil insert). Set conversationId. Call `fetchMessages()`.
    - `fetchMessages() async`: Fetch from chat_messages where conversation_id, order by created_at descending, limit 50. Reverse for display (oldest first). Populate `messages` and `knownMessageIds`. Set `hasMoreMessages = count == 50`. Set `nextCursor` from last item.
    - `fetchMoreMessages() async`: Same as above but with `.lt("created_at", value: nextCursor!)`. Prepend to messages array.
    - `fetchMessagesSince(_ timestamp: String) async`: Fetch messages with `.gt("created_at", value: timestamp)`. Deduplicate by checking knownMessageIds before appending. This is the catch-up-after-reconnect method.
    - `handleNewMessage(_ message: ChatMessage)`: Check `knownMessageIds` for dedup. If new, append to messages, add ID to known set. This is called by RealtimeManager's onNewMessage callback.
    - `sendMessage(text: String) async`: Create PendingMessage with .sending status, append to pendingMessages. Try to insert via Supabase REST (use auth.uid() as sender_id -- CRITICAL for RLS). On success, remove from pendingMessages. On failure, mark status .failed. Fire-and-forget call to chat-notification edge function after successful insert.
    - `retryMessage(_ pending: PendingMessage) async`: Remove from pending, re-send.
    - `markMessagesAsRead() async`: Call `supabase.rpc("mark_messages_read", params: ["p_conversation_id": conversationId, "p_user_role": "client"])`. Fire-and-forget.
    - `lastMessageTimestamp: String?`: Return `messages.last?.createdAt` -- used for catch-up queries.

    **DataManager.swift** -- Add `unreadChatCount: Int = 0` property. In `loadAllData()`, add a fetch for chat_conversations unread_count_client (single query, filter by client_id, read unread_count_client). This drives the badge on the chat tab.
