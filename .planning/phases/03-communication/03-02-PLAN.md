  <action>
    **Create an AttachmentData struct** (can live in ChatModels.swift or in AttachmentPickerSheet.swift):
    - `fileName: String`, `fileData: Data`, `mimeType: String`

    **AttachmentPickerSheet.swift** -- Bottom sheet with 3 options:
    1. **Photo Library**: Use SwiftUI `PhotosPicker` (import PhotosUI). Configure for images only (`.images` matching). On selection, use `loadTransferable(type: Data.self)` to get image data. Always convert to JPEG via `UIImage(data:)?.jpegData(compressionQuality: 0.8)` to normalize format (HEIC -> JPEG). Create AttachmentData with mimeType "image/jpeg".
    2. **Camera**: Use UIViewControllerRepresentable wrapping `UIImagePickerController(sourceType: .camera)`. Present via `.fullScreenCover()`. On capture, convert to JPEG. Create AttachmentData. Only show camera option if `UIImagePickerController.isSourceTypeAvailable(.camera)` (false on simulator).
    3. **File (PDF)**: Use `.fileImporter(isPresented:allowedContentTypes:)` with `UTType.pdf`. On selection, read data from URL. Create AttachmentData with mimeType "application/pdf".

    Present as a sheet with 3 large buttons in a vertical list (icon + label for each: "Photo Library", "Camera", "Choose File"). Dark theme styling matching app design system. Dismiss on selection.

    Callback: `onAttachmentSelected: (AttachmentData) -> Void`

    **ChatViewModel.swift** -- Extend with attachment support:
    - Add `uploadAttachment(_ data: AttachmentData) async throws -> (url: String, type: String, name: String)`:
      - Generate unique path: `"attachments/\(Int(Date().timeIntervalSince1970))-\(UUID().uuidString.prefix(7)).\(ext)"`
      - Compress images: if mimeType starts with "image/", compress via `UIImage(data:)?.jpegData(compressionQuality:)` targeting < 1MB (start at 0.8, reduce by 0.1 until under limit or quality < 0.1)
      - Upload: `supabase.storage.from("chat-attachments").upload(path:file:options: FileOptions(cacheControl: "3600", contentType: mimeType))`
      - Get public URL: `supabase.storage.from("chat-attachments").getPublicURL(path:)`
      - Return (url, type: image/file, name)
    - Modify `sendMessage` to accept optional `attachment: AttachmentData?`:
      - If attachment present: upload FIRST, then insert message with attachment_url, attachment_type, attachment_name
      - If upload fails: mark pending as .failed, do NOT send message
      - 10MB file size limit: check `data.fileData.count > 10_000_000` before upload, show error
    - Add `triggerLinkPreview(messageId: String, text: String) async`:
      - Detect URLs in text using `NSDataDetector(types: NSTextCheckingResult.CheckingType.link.rawValue)`
      - If URL found, call `supabase.functions.invoke("fetch-link-preview", options: .init(body: ["url": url, "messageId": messageId]))` (fire-and-forget)
      - The edge function caches the preview to the message's link_preview column, which Realtime will broadcast as an UPDATE

    **ChatInputBar.swift** -- Wire the attachment button:
    - Add `@State private var showAttachmentPicker = false`
    - Attachment button (plus.circle.fill) sets `showAttachmentPicker = true`
    - `.sheet(isPresented: $showAttachmentPicker) { AttachmentPickerSheet(onAttachmentSelected:) }`
    - Add `onAttachmentSelected: (AttachmentData) -> Void` callback parameter
    - When attachment is selected, call the callback (which ChatView will route to viewModel.sendMessage with attachment)

    **project.pbxproj** -- Add AttachmentPickerSheet.swift to project target.

    **Info.plist considerations**: Camera usage requires NSCameraUsageDescription. Photo library does NOT require a key when using PhotosPicker (it runs in a separate process). Add NSCameraUsageDescription = "Take photos to send in chat" to Info.plist if not already present.
  </action>

    **LinkPreviewCard.swift** -- OG metadata card:
    - Accept `preview: LinkPreviewData`
    - Card layout (matching iMessage/Slack style):
      - Thumbnail image (if preview.image exists): AsyncImage, 60px tall, full width, clipped to top corners
      - Site name (if exists): small text in AppColors.textTertiary (AppTypography.captionSmall)
      - Title: one line, bold text (AppTypography.bodyBold or .headingSmall) in AppColors.textPrimary
      - Description: up to 2 lines, text in AppColors.textSecondary (AppTypography.caption)
    - Background: AppColors.elevatedSurface with corner radius 12
    - Border: subtle white border at 0.05 opacity
    - Tappable: opens URL in Safari via `UIApplication.shared.open(url)`
    - Max width matches the message bubble width

    **MessageBubbleView.swift** -- Extend to handle attachments and link previews:

      - Show inline image preview using AsyncImage with the attachment URL

      - Rounded corners matching bubble shape
      - Tappable: `@State private var showImagePreview = false`, present ImagePreviewView via `.fullScreenCover`
