CRITICAL ORDER: Auth data MUST be imported before public data. Multiple public tables have foreign keys referencing auth.users(id). Importing public data first will cause FK constraint violations.

CRITICAL SETTING: ALL data imports MUST use `SET session_replication_role = replica`. This disables ALL triggers during import. Without this:
- The `on_auth_user_created` trigger fires for each auth.users row, creating duplicate profiles/user_roles
- Audit triggers fire during data import, creating spurious audit records
- Encryption triggers may double-encrypt data

```bash
  --variable ON_ERROR_STOP=1 \
  --command 'SET session_replication_role = replica' \
```

**A. Row Count Comparison (all public tables)**

Run on source:
```bash
psql --dbname "$OLD_DB_URL" -c "
SELECT schemaname || '.' || relname AS table_name, n_live_tup AS row_count
FROM pg_stat_user_tables
WHERE schemaname = 'public'
ORDER BY relname;
" -o /tmp/source_counts.txt
```

Run on target:
```bash
psql --dbname "$NEW_DB_URL" -c "
SELECT schemaname || '.' || relname AS table_name, n_live_tup AS row_count
FROM pg_stat_user_tables
WHERE schemaname = 'public'
ORDER BY relname;
" -o /tmp/target_counts.txt
```

NOTE: `pg_stat_user_tables.n_live_tup` may not be accurate immediately after COPY import. If counts show 0 on target, run `ANALYZE` first:
```bash
psql --dbname "$NEW_DB_URL" -c "ANALYZE;"
```
Then re-run the count query. If `n_live_tup` is still unreliable, use exact counts for critical tables:
```bash
psql --dbname "$NEW_DB_URL" -c "
SELECT 'clients' AS table_name, count(*) FROM clients
UNION ALL SELECT 'billing_records', count(*) FROM billing_records
UNION ALL SELECT 'wallet_transactions', count(*) FROM wallet_transactions
UNION ALL SELECT 'client_wallets', count(*) FROM client_wallets
UNION ALL SELECT 'ad_spend_daily', count(*) FROM ad_spend_daily
UNION ALL SELECT 'leads', count(*) FROM leads
UNION ALL SELECT 'campaigns', count(*) FROM campaigns
UNION ALL SELECT 'chat_messages', count(*) FROM chat_messages
UNION ALL SELECT 'profiles', count(*) FROM profiles

"
```
