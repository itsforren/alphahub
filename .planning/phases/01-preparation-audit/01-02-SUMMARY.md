## Decisions Made

1. **ENCRYPTION_KEY is critical-path:** Same AES-GCM key must be used in new project. Value comes from `GHL_ENCRYPTION_KEY` in zprofile. If wrong, all GHL OAuth tokens and Plaid access tokens become unreadable -- clients must re-authenticate.

2. **GHL_REDIRECT_URI needs dual update:** Must update both the Supabase secret value AND the GHL Marketplace App configuration to point to new project URL.

3. **SLACK_CHAT_WEBHOOK_URL investigation needed:** Code uses two separate Slack webhook URLs (ads manager and chat channels) but zprofile has only one `SLACK_WEBHOOK_URL`. The chat channel URL must be retrieved from old Supabase secrets or Slack workspace.

4. **LOVABLE_API_KEY replacement:** 5 functions use Lovable AI gateway (standard OpenAI chat format). Replace with direct LLM API call (OpenAI/Google/OpenRouter) -- only base URL and API key change needed.

5. **Stripe webhook secrets will regenerate:** When new webhook endpoints are created in Stripe Dashboard (Phase 4), Stripe generates new signing secrets. Old values are irrelevant.

6. **Supabase link deferred:** config.toml has `schedule` keys incompatible with CLI v2.75.0. Link will work after removing these keys during Phase 3 (cron scheduling is handled via pg_cron, not config.toml).

7. **17 inbound endpoints need URL updates** across: Stripe (4), GHL (2), Lead sources (3), Onboarding (2), Prospects (2), Fathom (1), Slack interactive (1), MCP (1), Tracking (1).
