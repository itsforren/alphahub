**Part B: Build SECRETS.md**

Using the cloned repo from Plan 01-01, build the complete secrets inventory.

1. Extract all `Deno.env.get()` calls:
```bash
cd /Users/forren/workspace/copy-alphahub/alphahub-v2
grep -roh 'Deno\.env\.get("[^"]*")' supabase/functions/ | sed 's/Deno\.env\.get("//; s/")//' | sort -u
grep -roh "Deno\.env\.get('[^']*')" supabase/functions/ | sed "s/Deno\.env\.get('//; s/')//" | sort -u
```

2. For each secret, determine which function(s) use it:
```bash
# For each secret name, find which function directories contain it
for secret in $(combined_list); do
  echo "$secret: $(grep -rl "$secret" supabase/functions/ | sed 's|supabase/functions/||; s|/.*||' | sort -u | tr '\n' ', ')"
done
```

3. Cross-reference with `~/.zprofile` to determine value availability:
```bash
# Check each secret against zprofile exports
grep "^export" ~/.zprofile | sed 's/export //; s/=.*//' | sort
```

4. Create SECRETS.md with this table structure:

| Secret Name | Category | Used By (functions) | zprofile Match | Value Location | Status |
|-------------|----------|---------------------|----------------|----------------|--------|

- `NOT_NEEDED` -- Lovable-specific, not needed after migration

**Part C: Build WEBHOOKS.md**

Inventory every external URL that points into AlphaHub and must change when the Supabase project changes.

1. Find all edge functions with `verify_jwt = false` (these are webhook receivers):
```bash
grep -B1 'verify_jwt = false' alphahub-v2/supabase/config.toml | grep '\[functions\.' | sed 's/\[functions\.//; s/\]//'
```

2. Find webhook-named functions:
```bash
ls alphahub-v2/supabase/functions/ | grep -i "webhook\|hook\|callback"
```

3. Search for external service references in edge functions:
```bash
grep -rn "stripe\|ghl\|leadconnector\|fathom\|slack\|twilio\|google" alphahub-v2/supabase/functions/ --include="*.ts" -l
```

4. Check for MCP proxy references (MCP consumers that call into AlphaHub):
```bash
grep -rn "mcp" alphahub-v2/supabase/functions/ --include="*.ts" -l
```
