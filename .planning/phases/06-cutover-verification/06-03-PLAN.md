**Step 2: Verify the 16 primary MCP tools work** by calling the mcp-proxy directly with curl (since restarting Claude Code to pick up the config change is a user action):

```bash
# Test the 16 named MCP tools exposed by alphahub-mcp
MCP_SECRET="${MCP_PROXY_SECRET}"
NEW_URL="https://qcunascacayiiuufjtaq.supabase.co"
PASS=0
FAIL=0

for tool in list_clients get_client_detail search_clients get_billing_summary get_ad_spend_overview get_campaign_health get_lead_pipeline get_communications get_financial_projections get_alerts get_stripe_balance get_stripe_customers get_stripe_invoices get_stripe_subscriptions get_stripe_charges get_stripe_payouts; do
  response=$(curl -s -w "\n%{http_code}" -X POST \
    "${NEW_URL}/functions/v1/mcp-proxy" \
    -H "Content-Type: application/json" \
    -H "x-mcp-secret: ${MCP_SECRET}" \
    -d "{\"tool\": \"${tool}\", \"params\": {}}")
  http_code=$(echo "$response" | tail -1)
  if [ "$http_code" = "200" ]; then
    echo "PASS: ${tool} (HTTP ${http_code})"
    PASS=$((PASS + 1))
  else
    echo "FAIL: ${tool} (HTTP ${http_code})"
    FAIL=$((FAIL + 1))
  fi
done

echo ""
echo "Primary tools: ${PASS} passed, ${FAIL} failed"
```

All 16 primary tools should return HTTP 200. If any return errors, investigate and fix.

**Step 3: Discover and verify ALL tools** via list_tools. The ROADMAP success criterion says "all 52 tools respond correctly" but the actual count is whatever list_tools returns (~50). Rather than assuming 52, dynamically iterate all tools:

```bash
MCP_SECRET="${MCP_PROXY_SECRET}"
NEW_URL="https://qcunascacayiiuufjtaq.supabase.co"

# Get the full list of available tools
ALL_TOOLS=$(curl -s -X POST "${NEW_URL}/functions/v1/mcp-proxy" \
  -H "Content-Type: application/json" \
  -H "x-mcp-secret: ${MCP_SECRET}" \
  -d '{"tool": "list_tools", "params": {}}' | jq -r '.result[].name')

TOTAL=$(echo "$ALL_TOOLS" | wc -l | tr -d ' ')
echo "Total tools reported by list_tools: ${TOTAL}"

PASS=0
FAIL=0
FAIL_LIST=""

# Iterate all tools and call each with empty params to verify it responds (HTTP 200)
for tool in $ALL_TOOLS; do
  # Skip list_tools itself and meta-tools that require specific params
  http_code=$(curl -s -o /dev/null -w "%{http_code}" -X POST \
    "${NEW_URL}/functions/v1/mcp-proxy" \
    -H "Content-Type: application/json" \
    -H "x-mcp-secret: ${MCP_SECRET}" \
    -d "{\"tool\": \"${tool}\", \"params\": {}}")
  if [ "$http_code" = "200" ]; then
    PASS=$((PASS + 1))
  else
    echo "FAIL: ${tool} (HTTP ${http_code})"
    FAIL=$((FAIL + 1))
    FAIL_LIST="${FAIL_LIST} ${tool}"
  fi
done

echo ""
echo "=== All Tools Verification ==="
echo "Total: ${TOTAL} | Passed: ${PASS} | Failed: ${FAIL}"
if [ "$FAIL" -gt 0 ]; then
  echo "Failed tools: ${FAIL_LIST}"
fi
```

**Note:** Some tools may return HTTP 200 with an empty result (valid -- the tool works but has no data for empty params). The key check is that the tool does NOT return 404/500. Tools that require specific params (like `run_query`) may return a 400 with a descriptive error -- this is acceptable as it proves the tool is deployed and reachable. Record the total count and pass/fail breakdown in the summary.
