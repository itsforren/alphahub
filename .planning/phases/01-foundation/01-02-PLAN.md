    Create the three core auth files:

    **SupabaseConfig.swift** — Singleton Supabase client with KeychainLocalStorage:
    ```swift
    import Supabase

    enum SupabaseConfig {
        static let client = SupabaseClient(
            supabaseURL: URL(string: "https://qcunascacayiiuufjtaq.supabase.co")!,
            supabaseKey: "<ANON_KEY>",  // Read from .env or hardcode anon key (public, safe to embed)
            options: SupabaseClientOptions(
                auth: .init(
                    storage: KeychainLocalStorage(
                        service: "com.alphaagent.ios",
                        accessGroup: nil
                    ),
                    flowType: .pkce
                )
            )
        )
    }
    ```
    **CRITICAL:** The anon key is a publishable key (safe to embed in client apps). Get it from the existing web app's `.env` file: `VITE_SUPABASE_PUBLISHABLE_KEY`. If not available, use a placeholder and mark for user to fill in.

    **AuthManager.swift** — @Observable class managing all auth state:
    - Properties: `session: Session?`, `isAuthenticated: Bool`, `isLoading: Bool`, `userRole: UserRole` (enum: client, admin, member, guest, referrer)
    - `startListening()` — async for-await loop on `supabase.auth.authStateChanges`. Handle `.initialSession`, `.signedIn`, `.signedOut` events. On sign-in, fetch user role via `supabase.rpc("get_user_role", params: ["_user_id": userId])`. All state updates on `@MainActor`.
    - `signIn(email:password:)` — calls `supabase.auth.signIn(email:password:)`
    - `signOut()` — calls `supabase.auth.signOut()`, resets all state
    - `resetPassword(email:)` — calls `supabase.auth.resetPasswordForEmail(email)` for OTP flow
    - `verifyOTP(email:token:)` — calls `supabase.auth.verifyOTP(email:token:type:.recovery)`
    - `updatePassword(newPassword:)` — calls `supabase.auth.update(user: UserAttributes(password:))`

    Role enum:
    ```swift
    enum UserRole: String, Codable {
        case client, admin, member, guest, referrer
    }
    ```

    The existing web app uses `supabase.rpc('get_user_role', { _user_id: userId })` which returns the `app_role` enum. Mirror this exactly.

    **BiometricManager.swift** — @Observable class for Face ID / passcode gating:
    - Properties: `isUnlocked: Bool`, `lastAuthTime: Date?`
    - Grace period: 30 seconds — if `Date().timeIntervalSince(lastAuthTime) < 30`, auto-unlock
    - `authenticate()` — uses `LAContext` with `.deviceOwnerAuthentication` policy (biometric + passcode fallback). On success, set `isUnlocked = true` and record `lastAuthTime`. All on `@MainActor`.
    - `lock()` — sets `isUnlocked = false`
    - Use localized reason: "Unlock Alpha Hub to access your account"

    **AVOID:** Do NOT use `.deviceOwnerAuthenticationWithBiometrics` — that has no passcode fallback. Must use `.deviceOwnerAuthentication`.

    4. Verify `SupabaseConfig` uses `KeychainLocalStorage` with service "com.alphaagent.ios"

    **AlphaHubApp.swift** — Wire everything together:
    ```swift
    @main
    struct AlphaHubApp: App {
        @State private var authManager = AuthManager(supabase: SupabaseConfig.client)
        @State private var biometricManager = BiometricManager()

        var body: some Scene {
            WindowGroup {
                RootView()
                    .environment(authManager)
                    .environment(biometricManager)
                    .task {
                        await authManager.startListening()
                    }
            }
        }
    }

    struct RootView: View {
        @Environment(AuthManager.self) private var auth
        @Environment(BiometricManager.self) private var biometric

        var body: some View {
            Group {
                if auth.isLoading {
                    LaunchScreenView()  // Simple black screen with logo
                } else if !auth.isAuthenticated {
                    LoginView()
                } else if !biometric.isUnlocked {
                    BiometricLockView()
                } else {
                    // Placeholder until plan 01-03 builds the real tab shell
                    PlaceholderView()
                }
            }
            .privacyBlur()
            .preferredColorScheme(.dark)
        }
    }
    ```

    **LaunchScreenView** — simple inline view (black + logo + spinner) shown during `auth.isLoading`.

    - The `RootView` currently shows `PlaceholderView()` when authenticated + unlocked. Plan 01-03 will replace this with `MainTabView(role:)`.
